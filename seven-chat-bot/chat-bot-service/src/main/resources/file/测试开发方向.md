### 测试开发岗位介绍

在互联网公司中, 开发岗和测试岗, 都是核心技术岗. 只不过分工不同.

- 开发岗, 也就是程序员, 负责根据需求, 编写代码, 开发程序.

- 测试开发岗, 则是根据需求, 编写测试用例, 对程序功能进行验证.

之所以叫做 "测试开发" , 是因为在工作中需要接触大量的 "开发相关工作", 例如:

- 阅读并理解程序员编写的源代码

- 编写测试用例 (尤其是自动化测试用例)

- 搭建测试环境

- 构建测试平台

- ......

当然不同的公司中, "测试开发岗" 可能有不同的叫法, 比如 "测试岗", "QA" 等. 他们的工作内容基本一致.

综上, 测试开发岗位, 同样也是需要具备扎实的编程功底.

因此咱们的测试开发方向中, 大量的篇幅都是开发相关的知识.

对于测试相关的方法和工具, 也会重点讲解

#### 主线课程篇 -- Redis

1）什么是 Redis

Redis 是一个内存数据库，也是企业后端开发中常用组件之一。

相比于 MySQL 来说，虽然 MySQL 功能强大，但是数据是保存到硬盘上，而Redis 则是把数据保存到内存中，由于内存的访问速度远远高于硬盘，因此 Redis 的速度比 MySQL 快很多。

Redis 在后端开发中通常搭配 MySQL 等数据库使用，作为缓存。

#### 算法篇 -- 背景介绍

就业现状：

目前越是好的公司， 对于算法方面的考察就越深入。 仅仅学习数据结构中的算法， 不足以满足冲击好公司的需求。

>

> 
> 2) 算法题的难度可以设置的非常难 (甚至题目都难以读懂)， 这样就可以起到充分的筛选作用， 让优秀的同学和水平一般的同学拉开差距。

- 面试中， 即使笔试已经考察过算法了， 面试也很可能有现场手撕代码的题目。

> 原因：
> 
> 笔试中只能看到做题的结果， 面试中面试官更关注的是编写代码的过程。 能够看到该应聘者是否编码熟练， 编码风格是否合理， 调试经验是否丰富等。
