#### 主线课程篇 -- C++ 语言

1) C++ 是什么

C 语言基础上， 添加更多功能， 构建出能力更强的编程语言

**其核心核心特性有：**

1. **多范式支持**：
   
   1. **面向过程**：保留C语言的函数和结构化编程特性。
   
   2. **面向对象**：通过封装、继承、多态实现代码复用与模块化。
   
   3. **泛型编程**：利用模板实现类型安全的通用算法（如STL容器）。

2. **静态类型与编译执行**：代码在编译时进行类型检查，生成高效机器码。

3. **硬件级控制**：支持指针、内存管理等底层操作，适合系统级开发。

**多应用于系统软件开发**、**游戏开发**、**高性能计算（HPC）、跨平台库与工具**。

#### 算法篇 -- 背景介绍

就业现状：

目前越是好的公司， 对于算法方面的考察就越深入。 仅仅学习数据结构中的算法， 不足以满足冲击好公司的需求。

- 笔试中， 目前很多公司采取 "线上笔试" 的方式。 线上笔试中算法题占据 80% 以上的比重。

> 原因：
> 
> 1) 线上笔试由于不受到场地限制， 会吸引大量的同学来参加。
> 
> 算法题不受到编程语言的限制， 同时又可以借助算法平台自动批改， 自动计算成绩， 无疑大大降低了企业招聘的成本。
> 
> 2) 算法题的难度可以设置的非常难 (甚至题目都难以读懂)， 这样就可以起到充分的筛选作用， 让优秀的同学和水平一般的同学拉开差距。

- 面试中， 即使笔试已经考察过算法了， 面试也很可能有现场手撕代码的题目。

> 原因：
> 
> 笔试中只能看到做题的结果， 面试中面试官更关注的是编写代码的过程。 能够看到该应聘者是否编码熟练， 编码风格是否合理， 调试经验是否丰富等。

#### 就业项目篇 -- 整体介绍

1) 项目是什么

项目是一个功能相对完整的， 能解决实际问题的程序。

一个程序员在公司中的日常工作， 就是 "做项目"。

同时， 项目也是面试中考察的重点环节。 是应聘者技术能力的综合体现。

2) 项目类别
- **企业级开发挑战项目：**则是**引入丰富的技术点， 复杂业务逻辑， 企业级开发组件，较大难度的万行代码量级的项目，**主要是满足具有更多时间，学习主动性更强或者致力于挑战大厂的学员。

- **标准项目：** 即为**满足大部分学员能做出来为基本前提，以学员们尽快更新出简历为目的**，同时聚焦在若干个功能点或者技术点的项目，每个标准项目侧重点不同。在学完课程之后，可以立即投身实习，秋招，春招等，达到快速对学习成果进行检验的目的。

- **极简项目:** 进一步简化功能点和技术点, 简化业务逻辑, 简化代码量, **使基础相对薄弱，学习时间非常紧张的同学也能学习掌握的项目**
